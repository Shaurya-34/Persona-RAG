# RAG Persona Chatbot

This project is a complete end-to-end Retrieval-Augmented Generation (RAG) system and Persona Extractor built on a dataset of 11,000+ multi-turn conversations (100k+ messages). It was built to demonstrate advanced contextual segmentation and user intelligence, all running entirely on local algorithms without relying on external LLM APIs like OpenAI.

## 🚀 Features & Architecture

### 1. How Topic Changes Are Detected (Chronological Splitting)
To ensure the chatbot understands the *flow* of conversation over time, messages are strictly processed chronologically. **Topic drift** is detected mathematically:
- **Overlapping Sliding Windows:** The conversation is grouped into rolling windows of 10 messages with a stride of 5, enabling smooth overlap so topics aren't abruptly cut off.
- **TF-IDF Vectorization:** Each window is converted into a TF-IDF vector, ignoring common stop words.
- **Cosine Similarity Drift:** We calculate the cosine similarity between consecutive windows. If the similarity drops below a strict threshold (`0.15`), it indicates a shift in the topic.
- **Topic Boundaries:** When a shift is detected, a topic checkpoint is created.
- **Extractive Summarization:** For each topic segment, sentences are scored by their TF-IDF weights, and the top 3 most significant sentences are extracted to form a concise topic summary.

*In addition to topics, chronological checkpoints are forcefully created every 100 messages to serve as a baseline historical anchor.*

### 2. How Retrieval Works (RAG)
The retrieval engine combines high-level summaries with low-level conversation chunks for a holistic context window:
- **Embeddings:** All topic summaries, 100-message checkpoints, and fine-grained conversation chunks (every 5 messages) are embedded using the `sentence-transformers/all-MiniLM-L6-v2` model.
- **Vector Database (FAISS):** The embeddings are normalized and indexed into a `FAISS` `IndexFlatIP` (Inner Product) vector store, which ensures blazingly fast cosine-similarity search.
- **Query Execution:** When a user asks a question, their query is embedded, and the FAISS index retrieves the Top-K (k=8) most relevant vectors.
- **Context Synthesis:** The chatbot seamlessly returns a combined answer featuring the most relevant **Topic Summaries** alongside the specific **Message Chunks**, displaying the exact source message indices to the user.

### 3. User Persona Extractor
Independently of the search engine, a signal-based deterministic extractor (`persona_builder.py`) processes the chronological messages to extract a persona for the two dominant users.
- **Signal Counting:** Analyzes sleep, food, and exercise habits by counting specific keyword occurrences and pattern matches.
- **Regex Extraction:** Detects recurring life events (marriages, new jobs) or locations using strict regular expression constraints.
- **Communication Profiling:** Calculates average message length, question/exclamation ratios, and emoji density to statistically determine the tone and style.
- **Deterministic Logic:** No LLM is used for persona extraction; all traits are derived from frequency counters and threshold-based classification.

### 4. RAG Synthesis Layer
To ensure the chatbot is fully portable and deployable on standard cloud infrastructure (like Render or Railway), it uses a **deterministic template-based synthesis**. 

This layer:
- Merges high-level **Topic Summaries** for context.
- Injects low-level **Conversation Evidence** for grounding.
- Ensures every answer is directly traceable to the original dataset with message indices.

All reasoning and grounding context is generated by the deterministic local RAG pipeline first.

## 💻 Running the Code Locally

### Prerequisites
- Python 3.10+
- Place the `conversations.csv` file in the root directory.

### 1. Install Dependencies
```bash
pip install -r requirements.txt
```

### 2. Build the Index & Persona (Data Pipeline)
This step processes the CSV, detects topics, embeds the data into FAISS, and extracts the user persona.
```bash
python data_pipeline.py
python persona_builder.py
```
*Note: This will generate a `data_cache/` directory containing the FAISS index and JSON metadata.*

### 3. Start the Chatbot UI
```bash
python app.py
```
The application will start via Uvicorn. **Open your browser and navigate to:**
👉 `http://127.0.0.1:8000`

## ☁️ Deployment Instructions

This repository is ready to be hosted on platforms like **Render**, **Railway**, or **Heroku**.

1. Create a new Web Service and connect this repository.
2. **Build Command:** 
   ```bash
   pip install -r requirements.txt && python data_pipeline.py && python persona_builder.py
   ```
3. **Start Command:** 
   ```bash
   uvicorn app:app --host 0.0.0.0 --port $PORT
   ```

*(For a faster production build, you can commit the `data_cache/` folder to your repository so the FAISS index doesn't need to rebuild on every deployment).*
